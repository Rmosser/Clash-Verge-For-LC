# Clash Verge Web Smoke Checklist

适用场景：懒猫微服里的 Clash Verge Web 端回归测试。

## Settings

- 外链
  - `使用手册`、`Telegram`、`GitHub` 应在浏览器新标签页打开。
- Verge 基础设置
  - `复制环境变量类型` 应复制剪贴板并出现成功提示。
  - `启动脚本 -> 浏览` 应打开浏览器文件选择器；取消时无报错；选择文件后提示已应用。
- Verge 高级设置
  - `配置目录`、`内核目录`、`日志目录` 应复制路径，并明确提示是 Web 版降级行为。
  - `开发者工具` 应显式禁用，并提示改用浏览器 DevTools。
  - `轻量模式设置` 应为只读说明，不允许保存桌面轻量模式配置。
  - `导出诊断信息` 应触发浏览器下载，并提示已开始下载。
  - `复制 Verge 版本号` 应复制剪贴板并提示成功。
- 界面设置
  - `常规托盘图标`、`系统代理托盘图标`、`TUN 模式托盘图标` 的 `Browse` 应打开浏览器文件选择器。
  - 未配置图标时不得请求 `common-.ico`、`sysproxy-.ico`、`tun-.ico`。

## Home

- `IP 信息` 卡片成功时应展示国家、IP、ASN 和位置信息。
- `IP 信息` 失败时只能显示产品化错误文案，不能直接暴露原始异常串。
- `IP 信息 -> 重试` 应重新发起请求。
- 订阅更新时间必须落在当前时代的合理年份范围内，不能出现 `58164-08-08` 之类的异常时间。

## Proxies

- 代理组 `延迟测试` 应至少给出以下之一：
  - 成功延迟
  - 超时
  - 网络错误
  - 目标不可达
- 失败时应有可见提示，不能是静默无反馈。

## Unlock

- `测试全部` 完成后应对每个媒体项落地状态。
- 即便整体超时，也要把超时/失败信息写回卡片，并显示汇总提示。
- 单项重测失败时应更新该卡片状态，而不是保持 `Pending`。

## Console / Network

- `日志` 页面在宿主机 journal 非空时，首次进入无需等待实时流也必须能看到历史日志。
- `日志` 页面如果 `/api/logs` 实时流失败，必须明确显示降级提示，不能继续静默空白。
- Settings 页面不应再出现以下静态资源 404：
  - `common-.ico`
  - `sysproxy-.ico`
  - `tun-.ico`
- 不允许存在“用户点击后完全无 console、无 network、无 toast、无 UI 变化”的静默按钮。
