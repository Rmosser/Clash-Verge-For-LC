# Clash Verge Web Smoke Checklist

适用对象：做 Web 端回归的人。

## Settings

- `使用手册`、`Telegram`、`GitHub` 必须在新标签页打开
- `复制环境变量类型` 必须复制成功并出现提示
- `启动脚本 -> 浏览` 必须打开浏览器文件选择器；取消无报错；选中文件后有成功提示
- `配置目录`、`内核目录`、`日志目录` 必须复制路径，并明确提示这是 Web 版降级行为
- `开发者工具` 必须显式禁用，并提示改用浏览器 DevTools
- `轻量模式设置` 只能读，不能保存桌面轻量模式配置
- `导出诊断信息` 必须触发浏览器下载
- `复制 Verge 版本号` 必须复制成功
- 三个托盘图标的 `Browse` 必须打开浏览器文件选择器
- 未配置图标时不得请求 `common-.ico`、`sysproxy-.ico`、`tun-.ico`

## Home

- `IP 信息` 成功时展示国家、IP、ASN、位置
- `IP 信息` 失败时只能显示产品化错误文案，不能把原始异常串直接露出来
- `IP 信息 -> 重试` 必须重新发起请求
- 订阅更新时间必须落在合理年份范围内，不能出现异常时间

## Proxies

- 代理组 `延迟测试` 必须落到明确结果：成功延迟、超时、网络错误或目标不可达
- 失败时必须有可见提示，不能静默无反馈

## Unlock

- `测试全部` 完成后必须给每个媒体项写回最终状态
- 即使整体超时，也要把超时或失败信息写回卡片并显示汇总提示
- 单项重测失败时必须更新该卡片状态，不能一直停在 `Pending`

## Console / Network

- `日志` 页面在宿主机 journal 非空时，首次进入就能看到历史日志
- `/api/logs` 实时流失败时，页面必须显示降级提示，不能静默空白
- Settings 页面不允许再出现 `common-.ico`、`sysproxy-.ico`、`tun-.ico` 的 404
- 不允许存在“点击后无 console、无 network、无 toast、无 UI 变化”的静默按钮
